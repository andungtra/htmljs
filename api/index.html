Why HtmlJs

HtmlJs is the most powerful MVVM JavaScript framework. HtmlJs completely separates HTML vs JavaScript but keep them connected together.
Below are some reasons that you should check out HtmlJs.

  1. MVVM & OOP
    MVVM pattern allows you to work with data structure and business logic without doing anything with the UI.
    OOP allows you to create and package the most compilcated component, fully customizable. MVVM and OOP are combined into one beautiful solution.

  2. Performance
    Performance is outstanding with HtmlJs. Checkout benchmarks of KnockoutJs, Angular2, HtmlJs

  3. Flexible
    HtmlJs is a tiny however powerful framework. You have a full-fledged framework with ajax, routing, partial loading, script loading. You can write a component/class in HtmlJs and integrate with other popular framework like jQuery, Angular 2.

Get started with html.observable

var helloWorld = html.observable('Hello world!');

html('#hwInput').value(helloWorld);
html('#hwSpan').text(helloWorld);
html.observable will give you an observable object for all data type except for Array.
Whenever you change its value inside, all of subscribed element will be update.
The change can be made by changing input value or change by code.
You can try to modify initial value inside it to see what happen. e.g:
var helloWorld = html.observable('Hello your name!');
Line number 3 html('#hwInput'').value(helloWorld) will query the DOM element and bind data to that element.
Line number 4 html('#hwSpan').text(helloWorld) will query the span element and set the text Hello world! to that span tag.
Now, you can change the code a little bit to see what happen.

Try it Yourself »
Explanation

In the previous example, you've tried binding first name and last name.
What if you want to display full name. If you create another observer named fullName
It could lead to another headache whenever user changes firstName or lastName value.
To solve this problem, just pass a function into html.observable. 

                                      var fullName = html.observable(function(){
                                        return firstName.data + ' ' + lastName.data;

Now you're able to see the full name displayed in the end of paragraph.

Validation

Validation made simple with HtmlJs


  • Example
  • HTML
  • JavaScript
<input id="simpleValidation" placeholder="Enter your name" class="form-control input-w160" /> <br />
html.observable.prototype.required = function (message) { var self = this; self.rules.push(function() { var input = self.bindingNodes[0]; if (self.data == null || self.data === '') { self.isValid = false; $(input).after($('<span class="html-error">').text(message)); } else { self.isValid = true; $(input).parent().find('.html-error').remove(); } }); return self; }; var helloWorld = html.observable('Nhan Nguyen') .required('Name is required.'); // Validate on data change helloWorld.subscribe(function () { helloWorld.validate(); }); html('#simpleValidation').value(helloWorld); html('#validationSpan').text(helloWorld);
Try it Yourself »
Explanation
To create a validation rule and handling error, you have to add method to html.observable class.
Let say you have to add "required" rule, the code template as below 
html.observable.prototype.required = function (message) {
    // Validation rule implementation go here
};

In the method, you have to add a function to "rules" property. The function must check against some constraints, and set "isValid"" property. 
html.observable.prototype.required = function (message) {
    // Validation rule implementation go here
    var self = this;
    self.rules.push(function () {
        if (self.data == null || self.data === '') {
            self.isValid = false;
        } else {
            self.isValid = true;
        }
    });

    return self;
};
You can add a message handler after constraints checking, for example 
    // previous code
    self.isValid = false;
    var input = self.bindingNodes[0];
    $(input).after($('<span class="html-error">').text(message));
If you want to display a popup instead of a message after the input, then we can replace the code as following 
    // previous code
    self.isValid = false;
    var input = self.bindingNodes[0];
    input.title = "Error!";
    input.setAttribute('data-toggle', 'popover');
    input.setAttribute('data-content', message);
    $(input).popover('show');

<div id="SList"> <form class="form-inline" role="form"> <input id="newPhone" class="form-control input-w160" placeholder="New smart phone..." /> <button id="addMore" class="btn btnViolet">Add</button> </form> <span>Your smart phones:</span> <ul> <li>Sample smart phone.</li> <li>These samples will be removed when binding data.</li> </ul> </div>
/* Declare data for the view. * Declaration should be put in another file. */ function addSP(e) { e.preventDefault(); smartPhone.data !== '' && smartPhoneList.add(smartPhone.data); smartPhone.data = ''; }; var smartPhone = html.observable(''); var smartPhoneList = html.observableArray([ 'Blackberry', 'iPhone', 'Lg g pro2' ]); /* End data declaration */ /* Binding data */ html('#SList ul').each(smartPhoneList, function(phone, index) { html.li.text(phone); }); html('#SList input').value(smartPhone); html('#SList button').onClick(addSP); /* End binding data */
Try it Yourself »
Explanation
Firstly, declare view model for the page 
var smartPhone = html.observable('');
var smartPhoneList = html.observableArray([
  'Blackberry',
  'iPhone',
  'Lg g pro2'
]);
smartPhone variable will be bounded to the textbox
smartPhoneList will be bounded to the list of smart phones.
Secondly render View from View-Model. 
html('#SList ul').each(smartPhoneList, function(phone, index){
    html.li.text(phone);
});
When you want to render a list, you take a parent element, in this example it is ul node
Then use each method. This method takes 2 parameters:
1. An array or an observed array (smartPhoneList in this case).
2. A callback function to render the list.
The callback function takes 2 parameters:
2.a. An item in the list
2.b. Item index
NOTE: You can't use each method without a parent.

Great! Now you will want to add more smart phone into the list. So bind the value of the input as below. 
html('#SList input').value(smartPhone);
Ok now, but it doesn't work because you will need to add an behaviour for Add button. 
var addSP = function(e){
  smartPhone.data !== '' && smartPhoneList.add(smartPhone.data);
  smartPhone.data = '';}
And bind the event click. This function add the current smart phone to the list and clear data of the input. 
html('#SList button').click(addSP)
Done! You can add a new one into smart phone list.

You may wonder why do need to pass extra child parameter to click event.
You're not forced to put that parameter in case you know what object you want to work with.

jQuery Integration

Integrate easily with any jQuery controls.

  • HTML
  • JavaScript
<input id="datepicker" class="form-control input-w160" placeholder="Select date" /> <br /> <span id="dateStr"></span>
/* CUSTOM CONTROL */ html.datepicker = function(observedDate) { // Get the current element of HtmlJs. var currentElem = this.context; // Bind that element to bootstrap datepicker var datepicker = $(currentElem).attr('readonly', true).css({cursor: 'pointer'}).datepicker({format:'dd/mm/yyyy'}) // Register change event to update observedDate datepicker.on('changeDate', function (e){ observedDate.data = $(currentElem).data('datepicker').date; }); //subscribe to observedDate //so that we can change the datepicker control by setting observerdDate's value observedDate.subscribe(function(val) { var widget = $(currentElem).data("datepicker"); if (widget) { widget.date = val; if (widget.date) { widget.setValue(widget.date); } } }); //return this (html object) for fluent API return this; }; /* END CUSTOM CONTROL */ var date = html.observable(new Date()); html('#datepicker').datepicker(date); html('#dateStr').text(date); date(new Date(2014, 0, 1));
Try it Yourself »
Explanation Firstly, you need to create a new control - html.datepicker
Inside that function, you also need register an event to update observer's value.
In this case, that event is 'changeDate'.
NOTE: you need to return this at the end to facilitate fluent API. 
html.datepicker = function(observedDate) {
    //get the current element of HtmlJs.
    var currentElem = this.element();

    //bind that element to bootstrap datepicker
    var datepicker = $(currentElem).datepicker({format:'dd/mm/yyyy'})

    //register change event to update observedDate
    datepicker.on('changeDate', function(e){
        observedDate($(currentElem).data('datepicker').date);
    });

    //return this (html object) for fluent API
    return this;
};
Secondly, you need to subscribe a function how to update your custom control. 
    //register change event to update observedDate
    //datepicker.on('changeDate', function(e){
    //    observedDate($(currentElem).data('datepicker').date);
    //});
    //...previous code

    //subscribe to observedDate
    //so that we can change the datepicker control by setting observerdDate's value
    observedDate.subscribe(function(val) {
        var widget = $(currentElem).data("datepicker");
        if (widget) {
            widget.date = val;
            if (widget.date) {
                widget.setValue(widget.date);
            }
        }
    });
After all declaration, you can use your custom control the same way as built-in controls.
Actually, all of built-in controls have the same structure like that.
Remember 4 steps:
First extend html object
Second get the DOM element and bind event to that element. This step will make change to observer
Third update DOM element whenever observer's value changed
Last (Optional) return this for fluent API

Ajax

Html Ajax implements using Promise pattern. You can set done and fail functions to the promise. Refer to Ajax API for more details. 
var ajax = html.ajax('someURL', {someParam: 'myData'}, 'GET', true);
Then you can use like this. 
ajax.done(function (data) {
	console.log(data);
}).fail(function (failReason) {
	console.log(failReason);
});
Usually, you'll use html.getJSON or html.postJSON
They're shorthand for html.ajax. In fact they use html.ajax with default parameter (POST/GET) 
var ajax = html.getJSON('myURL', {myParam: 'myData'});
	var ajax = html.getJSON('myURL', {myParam: 'myData'});
Another version for ajax is partial view. This method will load a view and append to a container. 
html('#container').partial('childView.html').done(function () {
    console.log('Partial loaded');
};
